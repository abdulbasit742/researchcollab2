# RCollab Canonical User Journeys

> One mental model per user type. No fragmentation.

---

## 🧑‍💼 Journey 1: Individual Professional

### Entry Point
- Sign up → Role selection → Profile creation → First verification

### Daily Usage Loop
```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  Check Opportunities → Execute Work → Log Outcomes  │
│         ↑                                    ↓      │
│         └──── Build Trust ← Get Paid ←──────┘      │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Progression Milestones
1. **Emerging** (0-30 days): Complete profile, first outcome
2. **Establishing** (30-90 days): 3+ verified outcomes, trust > 50
3. **Proven** (90-180 days): Consistent delivery, trust > 70
4. **Trusted** (180+ days): Network effects, opportunity flow
5. **Exemplary** (1+ year): Knowledge contribution, mentorship

### Key Surfaces
- `/home` - Daily command center
- `/opportunities` - Active matching
- `/progress` - Career trajectory
- `/career` - AI copilot conversations

### Exit & Re-entry
- Pause mode: Trust decay slowed, availability hidden
- Re-entry: Recovery roadmap, trust stabilization period

## 🔄 Universal Principles

### One Screen Per Intent
- Discovery → `/opportunities`
- Execution → `/deals`
- Reflection → `/progress`
- Growth → `/career`

### Progressive Disclosure
| User Tier | Visible Systems |
|-----------|-----------------|
| New | Core only (Layers 1-3) |
| Established | Core + Extended |
| Power | All individual systems |
| Institutional | Full platform |

### No Dead Ends
Every screen must answer:
- What can I do here?
- What should I do next?
- How do I get help?
