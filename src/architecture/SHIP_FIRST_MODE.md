## ✅ End-to-End Work Validation Checklist

A user MUST be able to:

- [ ] Discover an opportunity (matching engine)
- [ ] Communicate with counterparty (messaging)
- [ ] Agree on scope (deal creation)
- [ ] Execute milestones (deal room)
- [ ] Handle disputes (resolution flow)
- [ ] Close outcome (completion)
- [ ] See trust update (feedback loop)

**No admin intervention required.**

## 🧠 Build Discipline Rules

1. **No new system unless one is removed**
2. **No new UI without deleting another**
3. **No feature without daily use-case**
4. **No abstraction without user-facing benefit**

### Ship Confidence Checklist

Before marking anything "done":

- [ ] Understandable without explanation
- [ ] Does not rely on future systems
- [ ] Creates immediate value
- [ ] Does not increase cognitive load

If any fail → **cut or park**
