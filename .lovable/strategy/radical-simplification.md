# RCollab — Radical Product Simplification Protocol

## Core Mandate
Remove 60% of non-essential complexity. Protect escrow + ledger + institutional embedding + revenue clarity.

## The 5 Core Pillars (Everything Else Is Optional)
1. Escrow Engine
2. Immutable Ledger
3. Deal + Milestone Lifecycle
4. Institutional Dashboard
5. Sponsor Dashboard

## What To Remove Or Delay
- Advanced AI automation layers
- Complex predictive analytics
- National reporting dashboards
- Public ranking systems
- Cross-region benchmarking
- Government expansion layers
- Multi-layered badge systems
- Overengineered gamification
- Feature bloat in profiles
- Excessive subscription tiers (keep 2 max)
- Advanced liquidity models
- Speculative financial layers
- Premium analytics upsells
- Badge monetization
- Featured listings
- Data monetization

**Rule:** If it doesn't directly strengthen escrow-backed execution → remove.

## Simplified UX Per Role
- **Student:** View project → Submit milestone → View escrow status
- **Sponsor:** Create project → Fund escrow → Approve milestone → View progress
- **Faculty:** Approve project → Monitor progress → Review deliverables
- **Institution:** View dashboard → Export report

## Simplified Revenue (2 streams only)
1. Escrow transaction fee
2. Institutional subscription (2 tiers max)

## Simplified GTM
Target ONLY: Engineering/Tech universities. No business schools, no arts, no global push.

## Simplified Metrics (7 only)
1. Escrow reliability
2. Completion rate
3. Dispute rate
4. Institutional retention
5. Sponsor repeat rate
6. GMV
7. ARR

## Simplified Org
Financial Core Team, Institutional Partnerships, Sponsor Partnerships, Support, Compliance.

## Global Expansion: PAUSED
Until: 90% institutional retention, 12 months stability, positive cash flow.

## Internal Mantra
"We are escrow-backed academic execution infrastructure."
If a feature does not strengthen that sentence, do not build it.

## End State
Boring. Reliable. Clean. Focused. Trustworthy. Narrow but deep.
