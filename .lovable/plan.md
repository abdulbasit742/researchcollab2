

# Enable Google Sign-In for All Gmail/Google Account Holders

Currently, the Google sign-in button exists on the Auth page but uses an older integration method. We need to update it to use the Lovable Cloud managed Google OAuth, which works out of the box for any Gmail or Google account holder -- no extra configuration needed.

## What Changes

### 1. Configure Google Social Login
- Use the Lovable Cloud social auth configuration tool to generate the managed Google OAuth module at `src/integrations/lovable/`
- This provides a `lovable.auth.signInWithOAuth("google", ...)` function that handles everything automatically

### 2. Update `src/contexts/AuthContext.tsx`
- Replace the current `supabase.auth.signInWithOAuth({ provider: "google" })` call with `lovable.auth.signInWithOAuth("google", { redirect_uri: window.location.origin })`
- Import the lovable module from `src/integrations/lovable`

### 3. Update `vite.config.ts` (PWA fix)
- Add `/~oauth` to the service worker's `navigateFallbackDenylist` so OAuth redirect callbacks are never cached by the PWA service worker

## Result
Any Gmail or Google account holder will be able to sign up or sign in with one click using the "Continue with Google" button on the `/auth` page. No email verification step needed for Google accounts.

## Files

| File | Action |
|------|--------|
| `src/integrations/lovable/` | Auto-generated by configure tool |
| `src/contexts/AuthContext.tsx` | Edit `signInWithGoogle` method |
| `vite.config.ts` | Edit PWA config for OAuth route |
